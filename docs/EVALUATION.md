# Evaluation Report

## Current Evaluation Scope

This repository currently verifies the complete application pipeline in demo mode:

- image validation
- async analysis job creation
- status polling
- SQLite persistence
- history retrieval
- Grad-CAM-style heatmap file generation
- frontend API polling and UI states

Demo mode uses deterministic simulated predictions. It is useful for reviewing product behavior, but it is not a model-quality evaluation.

## Real Model Evaluation Status

Real inference mode loads EfficientNet-B4 DeepShield weights from Hugging Face. Before publishing model-quality claims, the repo should include a reproducible evaluation script and a dataset manifest.

Do not claim accuracy, precision, recall, F1, or AUC until all of these exist:

- fixed model weight hash
- fixed test dataset name/version
- train/validation/test split description
- preprocessing script
- evaluation command
- generated metrics artifact committed under `docs/results/`

## Required Metrics For Real Mode

When the real evaluation is added, report:

| Metric | Why it matters |
|---|---|
| Accuracy | Overall correctness |
| Precision on AI-generated class | Avoids falsely accusing real images |
| Recall on AI-generated class | Measures how many generated images are caught |
| F1 on AI-generated class | Balances precision and recall |
| AUC-ROC | Threshold-independent separation quality |
| Confusion matrix | Shows failure modes directly |

## Demo Mode vs Real Mode

| Mode | Requires weights | Prediction type | Intended use |
|---|---:|---|---|
| Demo mode | No | deterministic simulation | review, CI, frontend/backend flow |
| Real mode | Yes | EfficientNet-B4 inference | model demonstration and future evaluation |

## Known Limitations

- Demo-mode results are not forensic predictions.
- Real model performance may degrade on generators not represented in the evaluation set.
- Grad-CAM heatmaps are approximate explanations, not proof of manipulation.
- Highly compressed or adversarially edited images may reduce detector reliability.
- Image-generation models evolve quickly; evaluation should be repeated against newer generators.
